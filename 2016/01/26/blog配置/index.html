blog配置

2016.01.26 Mr.Q  热度 ℃ 条评论

主要配置文件：

  • 在根目录下的
    _config.yml (主配置)
    themes => theme => _config.yml (theme配置)
    source => _posts (md文件存放)

用到的git命令

  • 生成新的md文件： hexo n 'filename'

  • 整合静态文件(生成public文件夹)： hexo g

  • 本地预览： hexo server

  • 终止预览： Ctrl + C

  • 部署到远端仓库： hexo deploy

  • 远端不显示可键入(命令删除public)再g d: hexo clean

踩到的坑：

  • 访问路径中文报错
    $ hexo s
    然后本地访问报错The header content contains invalid characters
    解决：
    node_modules/hexo-server/lib/middlewares/route.js第27行的SetHeader代码。对重定向的url进行encode转码

  • 格式问题
    一定要注意xml配置格式，有空格，然后别的照着弄就好了

  • md文件头设置评论与否(注意空格)：
    comments: false

  • 端口被占用
    如果您想要更改端口，或是在执行时遇到了 EADDRINUSE 错误，可以在执行时使用 -p 选项指定其他端口，如下：
    hexo server -p 5000

  • 时不时提交登录一下：
    当你ssh密钥等都认为弄好了后，以下命令有所帮助
    git init 弹出登录一次后估计就能保存了

如果还解决不了，砸电脑吧
