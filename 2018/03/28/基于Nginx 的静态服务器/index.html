安装Nginx

系纺是Ubuntu Server 14.04.1 LTS 64位

配置.conf

新建.conf配置文件

PS-1：不同系统nginx.conf配置文件路径不一样，用以下命令查找
find / -name nginx.conf
PS-2：sudo是以root的权限进行修改（此时在tgq子用户下，当然一开始可以直接sudo su切换到root权限下操作，想换回tgq子用户就su tgq）
PS-3：这里新建的配置文件能够生效是因为在etc/nginx/nginx.conf下include了conf.d文件夹里面的所有后缀名为.conf文件

写入配置信息

Server配置区块

  1. 如果你想在一台服务器上运行多个网站的话， 你应该会用到这种配置区块。一般每个 server 区块的配置都可以单独放到一个文件里。比如一个test.conf配置文件里可以有多个server配置区块来创建不同的域名配置。

listen指令

listen 80：

  1. listen 指令可以设置服务器监听的端口号、IP 地址或者主机名。这里监听了 80 端口，这是 http 协议默认的端口号。
  2. 在监听的端口前面可以加上 IP 地址或许本地的主机名，比如：127.0.0.1:80 ，localhost:80，云主机公网IP:80。

server_name指令

server_name test.tgqnanman.com：
这个指令可以创建基于主机名的虚拟主机。比如上面的子域名例子，还有www.tgqnanman.com等等

root指令

root指令配置了这个虚拟主机的根目录，定义路径下默认访问的文件名。之前安装好 nginx，在浏览器里打开服务器的 IP 地址，看到的测试页面，就在这个目录的下面。 (注意：在centos系统安装的nginx欢迎页面在这个文件夹里/usr/share/nginx/html)。

index指令

index index.html index.htm：
index 指令设置了当请求的地址里不包含特定的文件的时候，默认打开的文件。这里设置成了 index.html index.htm ， 如果目录下面有 index.html 就打开它，如果没有就去找 index.htm。
